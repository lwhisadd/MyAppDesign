## ⚠️ 重要閱讀說明（給 AI Agent 與設計人員）

> **本 Repository 的每一個根目錄，均代表一個獨立的技術討論集合。**
>
> 各目錄之間**沒有任何繼承、依賴或延伸關係**，不可跨目錄合併判斷、交叉引用，或將不同目錄的技術決策視為同一套方案的組成部分。

### 請務必遵守以下規則

- **單一目錄即為單一完整方案**：每個目錄內的文件共同構成一次完整的技術討論，應整體閱讀，不可拆散。
- **禁止跨目錄混合閱讀**：不同目錄的討論可能採用不同技術棧、不同架構決策，強行合併會產生矛盾與錯誤結論。
- **禁止跨目錄交叉引用**：引用技術決策時，必須明確指定所屬目錄，不可假設其他目錄的結論與本目錄一致。
- **目錄間無優先順序**：各目錄不代表版本迭代，不存在「舊版」與「新版」的關係，除非目錄內文件明確說明。

## 給 AI Agent 的特別說明

本 Repository 供多輪、多主題的技術設計討論使用。若你是 AI Agent，請注意：

1. **Context 邊界**：每次閱讀請以單一目錄為 context 邊界，不要將多個目錄的內容混入同一個 context window 進行推論。
2. **技術決策不可遷移**：A 目錄決定使用某技術，不代表 B 目錄也適用，請勿自動套用。
3. **目錄內文件有閱讀順序**：若目錄內存在多份文件（如 `討論1.md`、`討論2.md`），請依檔名數字順序閱讀，後者可能修正前者的決策。
4. **遇到衝突以目錄內最後一份文件為準**：同一目錄內若有矛盾，以編號最大的文件為最終決策。
