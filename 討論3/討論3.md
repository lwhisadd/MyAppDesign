## 三、各層技術選型

### 3-1 A 層：身份驗核

- 使用者登入請求先送至 **EasyAPI** 進行身份驗核
- EasyAPI 回傳驗核成功結果 + 使用者帳密（或加密 token），供後續自動登入使用
- **帳密不在前端持有**，由後端安全傳遞給 Session 管理層

### 3-2 B+C 層：瀏覽器 Session 自動化（Playwright）

選用 **Playwright（Node.js）**：

- 支援 headless + headed 兩種模式
- 原生支援**多 BrowserContext**（每個 context = 獨立 Session，互相隔離）
- 可直接操作表單填寫、按鈕點擊、等待頁面跳轉

```ts
// 建立獨立 Session 並完成登入
const browser = await chromium.launch({ headless: true })
const context = await browser.newContext() // 每個 context = 獨立 session
const page = await context.newPage()

await page.goto('http://10.10.10.1/login.php')
await page.fill('#username', credentials.username)
await page.fill('#password', credentials.password)
await page.click('button[type=submit]')
await page.waitForURL('**/menu.php') // 確認成功進入 menu.php
```

### 3-3 D 層：AI 語意解析

安裝於主機 `10.10.10.1`，選型對比：

| 方案 | 說明 | 適用情境 |
|------|------|---------|
| Ollama + Hermes-3 | 本地 LLM，提供 REST API | 無法連外網、要求低延遲 |
| OpenAI API | 雲端，語意理解能力強 | 可連外網 |
| OpenCode | 支援 function calling | 需要結構化工具調用 |

**語意解析目標**：將自然語言轉為結構化操作指令

```json
// 使用者說：「查詢 2026 年 6 月的報表」
{
  "action": "navigate",
  "target": "report",
  "params": { "year": 2026, "month": 6 }
}
```

### 3-4 E 層：多 Session 管理服務

這是本系統的核心，需要一個常駐服務管理所有使用者的 Playwright context：

```ts
// session-manager.ts
const sessions = new Map<string, {
  context: BrowserContext,
  page: Page,
  userId: string,
  createdAt: Date
}>()

async function getOrCreateSession(userId: string, credentials: Credentials) {
  if (sessions.has(userId)) return sessions.get(userId)!

  const context = await browser.newContext()
  const page = await context.newPage()
  await loginToSite(page, credentials)

  const session = { context, page, userId, createdAt: new Date() }
  sessions.set(userId, session)
  return session
}
```

**Session 生命週期管理需考量：**
- 閒置逾時自動釋放（建議 30 分鐘）
- 定期 heartbeat 維持 Session 存活
- 異常斷線自動重連並重新登入

## 四、推薦技術棧

| 層次 | 技術 | 理由 |
|------|------|------|
| Session 自動化 | **Playwright（Node.js）** | 多 context 隔離、穩定、headless 支援 |
| 多 Session 管理服務 | **Express + TypeScript** | 輕量 REST 服務，部署於 10.10.10.1:3002 |
| AI 語意解析 | **Ollama + Hermes-3** | 本地部署，支援 function calling |
| 身份驗核 | **EasyAPI**（既有） | 沿用，零額外成本 |
| 前端操作介面 | **Next.js（MyAppDesign）** | 沿用，透過 EasyAPI 通訊 |

## 五、完整訊息流程

```
[使用者瀏覽器]
  → POST /api/command { userId, naturalLanguage }
  → [MyAppDesign Next.js]
  → [EasyAPI 10.10.10.1:3001] 驗核身份
  → [Session Manager Service 10.10.10.1:3002]
      → Ollama/Hermes 語意解析 → 結構化指令
      → Playwright context（該使用者專屬）
          → http://10.10.10.1/login.php（若尚未登入則先登入）
          → menu.php 執行對應操作
      → 回傳操作結果 JSON
  → [MyAppDesign] 顯示結果給使用者
```

## 七、補強建議

### 必補

| 補強項目 | 建議技術 | 理由 |
|----------|----------|------|
| Session 持久化 | Redis | 服務重啟後 Session 狀態不遺失 |
| 操作指令 Queue | BullMQ | 避免多指令並發衝突 |
| 錯誤重試機制 | Playwright retry + EasyAPI fallback | 自動化操作不穩定性保護 |

### 視需求補

| 補強項目 | 建議技術 | 適用情境 |
|----------|----------|---------|
| Session 錄影除錯 | Playwright trace viewer | 自動化操作異常排查 |
| 操作結果推播 | WebSocket / SSE | 長時間操作進度即時回報 |
| 多智能體協作 | OpenAI Agents SDK | 更複雜的多步驟任務規劃 |
