## 二、主要問題與風險

### 2-1 SSR / Hydration 邊界（已確認解決方案）

shadcn/ui 使用 Tailwind CSS，完全不採用 CSS-in-JS，因此**不存在 Ant Design 的 FOUC 問題**。樣式在 build time 即生成靜態 CSS 檔，SSR 與 CSR 輸出天然一致。

但 Next.js App Router 的 hydration 四條規則仍完整適用（與討論1.md 相同）：

1. **Server Component 不放非確定性內容**（`new Date()`、`Math.random()`）
2. **瀏覽器 API 只在 Client Component 使用**（`window`、`localStorage`）
3. **DOM-heavy 元件以 `dynamic({ ssr: false })` 隔離**（圖表、地圖、編輯器）
4. **`suppressHydrationWarning` 只當最後手段**

### 2-2 資料層（已確認解決方案）

與討論1.md 完全相同，選用 **TanStack Query + Zustand** 兩層分離職責：

- **TanStack Query**：伺服器狀態（分頁、篩選、快取、Optimistic Update）
- **Zustand**：前端本地狀態（sidebar、modal、row 選取、UI 偏好）

### 2-3 Node.js 角色（已確認解決方案）

與討論1.md 完全相同，選用 **[EasyAPI](https://github.com/lwhisadd/EasyAPI) 作為唯一 REST API 接口**，Next.js 不使用 Route Handlers，所有 API 呼叫統一透過 `lib/api-client.ts` 指向 EasyAPI。

### 2-4 Dark Mode / 主題切換（已確認解決方案）

shadcn/ui 採用 CSS 變數系統定義 design tokens，主題切換透過在 `<html>` 切換 `class="dark"` 實現。選用 `next-themes` 作為主題管理層。

**問題根源**：若直接在 Client Component render 時讀取主題，Server 端與 Client 端初始輸出不一致，導致 hydration mismatch 與短暫閃爍。

**解決方案**：`next-themes` 透過在 `<html>` 注入 inline script 在 DOM paint 前設定主題 class，從根本消除 FOUC。

```tsx
// app/layout.tsx
import { ThemeProvider } from 'next-themes'

export default function RootLayout({ children }: React.PropsWithChildren) {
  return (
    <html lang="zh-TW" suppressHydrationWarning>
      <body>
        <ThemeProvider
          attribute="class"
          defaultTheme="system"
          enableSystem
          disableTransitionOnChange
        >
          {children}
        </ThemeProvider>
      </body>
    </html>
  )
}
```

**注意事項**：
- `<html>` 必須加 `suppressHydrationWarning`，因為 `next-themes` 會在 hydration 前修改 class，這是唯一合法使用 `suppressHydrationWarning` 的場景
- `disableTransitionOnChange` 可防止主題切換時 CSS transition 造成的顏色閃爍
- `tailwind.config.ts` 需設定 `darkMode: 'class'`

```ts
// tailwind.config.ts
const config = {
  darkMode: 'class',
  // ...
}
```

**主題切換元件：**

```tsx
// components/ui/theme-toggle.tsx
'use client'
import { useTheme } from 'next-themes'
import { Button } from '@/components/ui/button'
import { Moon, Sun } from 'lucide-react'

export function ThemeToggle() {
  const { theme, setTheme } = useTheme()
  return (
    <Button
      variant="ghost"
      size="icon"
      onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}
    >
      <Sun className="h-4 w-4 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
      <Moon className="absolute h-4 w-4 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
      <span className="sr-only">切換主題</span>
    </Button>
  )
}
```

### 2-5 打包效能（已確認解決方案）

shadcn/ui 的打包特性與 Ant Design 根本不同，主要問題已消除：

- **無 barrel export 問題**：元件為專案原始碼，非 npm 套件
- **Tailwind CSS v4 JIT**：build time 自動 purge 未使用的 class，CSS 輸出極小
- **Radix UI 按需引入**：各 primitive 為獨立套件（`@radix-ui/react-dialog` 等），天然 tree-shaking

## 三、SSR / Hydration 邊界詳細方案

### 3-1 shadcn/ui 與 Tailwind 的 SSR 優勢

shadcn/ui + Tailwind CSS 在 SSR 環境下天然安全：

- **無執行期樣式注入**：Tailwind 在 build time 生成靜態 `.css` 檔，Server 與 Client 輸出完全一致
- **無 CSS-in-JS runtime**：不需要 `@ant-design/nextjs-registry` 這類 SSR 樣式注入方案
- **CSS 變數主題**：shadcn/ui 使用 CSS 變數定義 design tokens，Server 端即可正確輸出

```css
/* globals.css — shadcn/ui CSS 變數定義 */
@layer base {
  :root {
    --background: 0 0% 100%;
    --foreground: 222.2 84% 4.9%;
    --primary: 222.2 47.4% 11.2%;
    --primary-foreground: 210 40% 98%;
    /* ... */
  }
  .dark {
    --background: 222.2 84% 4.9%;
    --foreground: 210 40% 98%;
    /* ... */
  }
}
```

### 3-2 避免 hydration mismatch 的四個規則

#### 規則 1：Server Component 不放非確定性內容

```tsx
// ❌ 錯誤
export default function Page() {
  return <div>{new Date().toLocaleString()}</div>
}

// ✅ 正確：改為 client component + useEffect
'use client'
import { useEffect, useState } from 'react'

export default function Clock() {
  const [time, setTime] = useState('')
  useEffect(() => {
    setTime(new Date().toLocaleString())
  }, [])
  return <div>{time}</div>
}
```

#### 規則 2：瀏覽器 API 只在 client 端使用

```tsx
'use client'
import { useEffect, useState } from 'react'

export function useIsClient() {
  const [isClient, setIsClient] = useState(false)
  useEffect(() => {
    setIsClient(true)
  }, [])
  return isClient
}
```

#### 規則 3：DOM-heavy 元件以 `dynamic({ ssr: false })` 隔離

```tsx
// components/client-only/ClientOnlyChart.tsx
'use client'
import dynamic from 'next/dynamic'
import { Skeleton } from '@/components/ui/skeleton'

const HeavyChart = dynamic(() => import('./HeavyChart'), {
  ssr: false,
  loading: () => <Skeleton className="h-64 w-full" />,
})

export default HeavyChart
```

#### 規則 4：`suppressHydrationWarning` 只當最後手段

唯一合法的常規使用場景是配合 `next-themes` 的 `<html>` 標籤（見 2-4）。

### 3-3 邊界切分原則

| 類型 | 建議放置位置 | 說明 |
|------|--------------|------|
| 靜態頁面框架 | Server Component | layout、page shell、SEO 內容 |
| API 預抓資料 | Server Component | 首屏資料可先預取 |
| 表格互動 | Client Component | pagination、sort、filter、selection |
| Modal / Sheet / Theme Switch | Client Component | 高互動 UI 狀態 |
| 第三方 DOM-heavy 套件 | Client-only island | 用 `dynamic({ ssr: false })` 隔離 |

**Optimistic Update：**

```ts
const mutation = useMutation({
  mutationFn: updateRecord,
  onMutate: async (newData) => {
    await queryClient.cancelQueries({ queryKey: ['table-data'] })
    const snapshot = queryClient.getQueryData(['table-data'])
    queryClient.setQueryData(['table-data'], (old) => patch(old, newData))
    return { snapshot }
  },
  onError: (_err, _vars, ctx) => {
    queryClient.setQueryData(['table-data'], ctx?.snapshot)
  },
  onSettled: () => queryClient.invalidateQueries({ queryKey: ['table-data'] }),
})
```

### 5-1 shadcn/ui DataTable 架構

shadcn/ui DataTable 是 **TanStack Table（headless 邏輯層）+ shadcn/ui Table 元件（樣式層）** 的組合。

**Column 定義（TypeScript 強型別）：**

```ts
// components/data-table/columns.tsx
import { ColumnDef } from '@tanstack/react-table'
import { Button } from '@/components/ui/button'
import { ArrowUpDown } from 'lucide-react'

export type Payment = {
  id: string
  amount: number
  status: 'pending' | 'processing' | 'success' | 'failed'
  email: string
}

export const columns: ColumnDef<Payment>[] = [
  {
    accessorKey: 'status',
    header: '狀態',
  },
  {
    accessorKey: 'email',
    header: ({ column }) => (
      <Button
        variant="ghost"
        onClick={() => column.toggleSorting(column.getIsSorted() === 'asc')}
      >
        Email
        <ArrowUpDown className="ml-2 h-4 w-4" />

  {
    accessorKey: 'amount',

      const formatted = new Intl.NumberFormat('zh-TW', {
```

**DataTable 通用封裝元件：**

```tsx
// components/data-table/data-table.tsx
'use client'
import {
  ColumnDef, flexRender, getCoreRowModel,
  getPaginationRowModel, getSortedRowModel,
  SortingState, useReactTable,
} from '@tanstack/react-table'
import {
  Table, TableBody, TableCell,
  TableHead, TableHeader, TableRow,
} from '@/components/ui/table'
import { Button } from '@/components/ui/button'
import { useState } from 'react'

interface DataTableProps<TData, TValue> {
  columns: ColumnDef<TData, TValue>[]
  data: TData[]
}

export function DataTable<TData, TValue>({ columns, data }: DataTableProps<TData, TValue>) {
  const [sorting, setSorting] = useState<SortingState>([])

  const table = useReactTable({
    data,
    columns,
    getCoreRowModel: getCoreRowModel(),
    getPaginationRowModel: getPaginationRowModel(),
    getSortedRowModel: getSortedRowModel(),
    onSortingChange: setSorting,
    state: { sorting },
  })

  return (
    <div>
      <div className="rounded-md border">
        <Table>
          <TableHeader>
            {table.getHeaderGroups().map((headerGroup) => (
              <TableRow key={headerGroup.id}>
                {headerGroup.headers.map((header) => (
                  <TableHead key={header.id}>
                    {header.isPlaceholder ? null :
                      flexRender(header.column.columnDef.header, header.getContext())}
                  </TableHead>
                ))}
              </TableRow>
            ))}
          </TableHeader>
          <TableBody>
            {table.getRowModel().rows?.length ? (
              table.getRowModel().rows.map((row) => (
                <TableRow key={row.id} data-state={row.getIsSelected() && 'selected'}>
                  {row.getVisibleCells().map((cell) => (
                    <TableCell key={cell.id}>
                      {flexRender(cell.column.columnDef.cell, cell.getContext())}
                    </TableCell>
                  ))}
                </TableRow>
              ))
            ) : (
              <TableRow>
                <TableCell colSpan={columns.length} className="h-24 text-center">
                  尚無資料
                </TableCell>
              </TableRow>
            )}
          </TableBody>
        </Table>
      </div>
      <div className="flex items-center justify-end space-x-2 py-4">
        <Button variant="outline" size="sm" onClick={() => table.previousPage()} disabled={!table.getCanPreviousPage()}>
          上一頁
        </Button>
        <Button variant="outline" size="sm" onClick={() => table.nextPage()} disabled={!table.getCanNextPage()}>
          下一頁
        </Button>
      </div>
    </div>
  )
}
```

```ts
export const userFormSchema = z.object({
```

```tsx
        <FormField
          control={form.control}
          name="name"
          render={({ field }) => (
            <FormItem>
              <FormLabel>姓名</FormLabel>
              <FormControl>

              </FormControl>
              <FormMessage />
            </FormItem>

        <FormField
          control={form.control}
          name="email"

              <FormLabel>Email</FormLabel>
              <FormControl>
                <Input placeholder="輸入 Email" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
```

## 七、後端 API 方案

與討論1.md 完全相同，直接沿用：

**選用 [EasyAPI](https://github.com/lwhisadd/EasyAPI) 作為唯一 REST API 接口**，以獨立服務模式運行，與 Next.js 分離於不同 port / process。

**整合架構：**

```
┌───────────────────────────────┐        HTTP / fetch        ┌──────────────────────────┐
│   MyAppDesign（Next.js）      │  ─────────────────────►   │   EasyAPI（Node.js）     │
│                               │                            │   獨立 port（如 3001）   │
│   TanStack Query queryFn      │  ◄─────────────────────   │   統一 REST API 接口     │
│   lib/api-client.ts → EasyAPI │        JSON Response       │   API Key 驗證           │
│   Next.js Route Handlers ❌   │                            └──────────────────────────┘
└───────────────────────────────┘
```

```ts
// lib/api-client.ts
const BASE_URL = process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:3001'

export async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${BASE_URL}${path}`, {
    ...init,
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': process.env.NEXT_PUBLIC_API_KEY ?? '',
      ...init?.headers,
    },
  })
  if (!res.ok) throw new Error(`API error: ${res.status}`)
  return res.json() as Promise<T>
}
```

## 八、補強建議

### 必補

| 補強項目 | 建議技術 | 理由 |
|----------|----------|------|
| 伺服器狀態 | `TanStack Query` | 分頁、快取、Optimistic Update 統一管理 |

### 視需求補

| 補強項目 | 建議技術 | 適用情境 |

## 十二、結論

1. **shadcn/ui 元件完全擁有**，無版本鎖定，可自由修改任何元件細節。
2. **Tailwind CSS 天然解決 SSR 樣式問題**，不需要 CSS-in-JS 注入方案。
3. **`next-themes` 負責 Dark Mode**，透過 inline script 在 paint 前設定主題，消除 FOUC。
4. **表格統一走 shadcn/ui DataTable**，基於 TanStack Table，功能完整且型別安全。
5. **表單統一走 React Hook Form + Zod**，shadcn/ui Form 元件天然整合，無需分層策略。
6. **TanStack Query + Zustand 狀態層完全沿用**，與 UI 元件庫無耦合。
7. **EasyAPI 後端架構完全沿用**，UI 替換對後端零影響。
8. **打包效能先天優於 Ant Design**，保留 Bundle Analyzer + CI 監控維持長期健康。

> 下一步建議：執行 `npx shadcn@latest init` 初始化專案，安裝 `next-themes`、`@tanstack/react-table`、`react-hook-form`、`zod`，建立 `components/ui/theme-toggle.tsx`、`components/data-table/data-table.tsx`、`components/providers.tsx`、`lib/api-client.ts` 四個基礎骨架，並設定 `NEXT_PUBLIC_API_URL` 環境變數指向 EasyAPI 服務。
