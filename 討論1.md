## 二、主要問題與風險

### 2-1 SSR / Hydration 邊界（已確認解決方案）

Next.js App Router 採用 Server Components + Client Components 混合模式，若 Server 與 Client 首次 render 的輸出不一致，就會出現 hydration mismatch。這個問題分成兩種：

1. **Ant Design 樣式首屏未注入（FOUC）**
2. **元件在 Server / Client 的 HTML 輸出不一致**

Ant Design 在 Next.js App Router 下需要 `@ant-design/nextjs-registry` 處理首屏樣式注入，否則會出現樣式閃爍（FOUC）。應在 `app/layout.tsx` 的最外層包覆 `AntdRegistry`。

### 2-2 資料層缺少（已確認解決方案）

選用 **TanStack Query + Zustand** 兩層分離職責：

- **TanStack Query**：負責一切來自 API 的伺服器狀態（分頁、篩選、快取、Optimistic Update）
- **Zustand**：負責純前端本地狀態（側邊欄開關、跨頁選取列、Modal 狀態、全局 UI 設定）

兩者不重疊、職責清晰，不需要 Redux。

### 2-3 Node.js 角色模糊

Node.js 若與 Next.js 同 repo，需明確定義其角色：
- **Next.js Route Handlers**（App Router 內建）
- **獨立 Express / Fastify 服務**（另一個 port / process）
- **BFF（Backend for Frontend）**

三者責任劃分不同，應在架構文件中明確記錄。

### 2-4 打包效能

Ant Design 元件數量多，若不做 tree-shaking，首屏 JS 體積偏重。建議搭配 `@ant-design/nextjs-registry` 與 Next.js 內建的 import optimization。

## 三、SSR / Hydration 邊界詳細方案

### 3-1 Ant Design 首屏樣式注入

Ant Design 使用 CSS-in-JS，若 SSR 階段沒有把樣式一起輸出到 HTML，客戶端 hydrate 前會先看到無樣式畫面。解法是使用 `@ant-design/nextjs-registry`。

```tsx
// app/layout.tsx
import { AntdRegistry } from '@ant-design/nextjs-registry'
import Providers from '@/components/Providers'

export default function RootLayout({ children }: React.PropsWithChildren) {
  return (
    <html lang="zh-TW">
      <body>
        <AntdRegistry>
          <Providers>{children}</Providers>
        </AntdRegistry>
      </body>
    </html>
  )
}
```

**原則：**
- `AntdRegistry` 放最外層
- `QueryClientProvider` 放在 `AntdRegistry` 內層
- 主題（theme）若會影響 SSR 輸出，應由 **cookie** 提供初始值，避免 server / client 主題不同

### 3-2 避免 hydration mismatch 的四個規則

#### 規則 1：Server Component 不要放非確定性內容

以下內容不要直接放在 Server Component 首次輸出：
- `new Date()`
- `Math.random()`
- 依賴瀏覽器尺寸的值
- client-only user preference

```tsx
// ❌ 錯誤
export default function Page() {
  return <div>{new Date().toLocaleString()}</div>
}

// ✅ 正確：改為 client component + useEffect
'use client'
import { useEffect, useState } from 'react'

export default function Clock() {
  const [time, setTime] = useState('')
  useEffect(() => {
    setTime(new Date().toLocaleString())
  }, [])
  return <div>{time}</div>
}
```

#### 規則 2：瀏覽器 API 只在 client 端使用

像 `window`、`document`、`localStorage`、`matchMedia` 都不能直接在 Server Component 中執行。

```tsx
'use client'
import { useEffect, useState } from 'react'

export function useIsClient() {
  const [isClient, setIsClient] = useState(false)
  useEffect(() => {
    setIsClient(true)
  }, [])
  return isClient
}
```

搭配方式：

```tsx
const isClient = useIsClient()
return isClient ? <ClientOnlyWidget /> : null
```

#### 規則 3：完全依賴 DOM 的元件以 `dynamic({ ssr: false })` 隔離

圖表、地圖、富文字編輯器、拖曳元件等常需要完整瀏覽器環境，應封裝成 client-only island。

```tsx
// components/ClientOnlyChart.tsx
'use client'
import dynamic from 'next/dynamic'
import { Skeleton } from 'antd'

const HeavyChart = dynamic(() => import('./HeavyChart'), {
  ssr: false,
  loading: () => <Skeleton active />,
})

export default HeavyChart
```

**原則：**
- `dynamic({ ssr: false })` 要寫在 `'use client'` 檔案內
- 搭配 loading placeholder，避免 CLS
- 不要把整頁都設成 client-only，只隔離必要元件

#### 規則 4：`suppressHydrationWarning` 只當最後手段

若是第三方嵌入內容、廣告腳本、無法完全控制的輸出差異，可局部使用：

```tsx
<div suppressHydrationWarning>
  {unavoidableDynamicContent}
</div>
```

但這不是真正修復，只是靜音警告，不應當作常規方案。

### 3-3 建議的邊界切分原則

| 類型 | 建議放置位置 | 說明 |
|------|--------------|------|
| 靜態頁面框架 | Server Component | layout、page shell、SEO 內容 |
| API 預抓資料 | Server Component 或 Route Handlers | 首屏資料可先預取 |
| 表格互動 | Client Component | pagination、sort、filter、selection |
| Modal / Drawer / Theme Switch | Client Component | 高互動 UI 狀態 |
| 第三方 DOM-heavy 套件 | Client-only island | 用 `dynamic({ ssr: false })` 隔離 |

## 四、資料層詳細方案

### 4-1 TanStack Query — 伺服器狀態

**職責**：所有來自 API 的資料（列表、詳情、分頁、搜尋結果）

**Next.js App Router 整合：**

```tsx
// app/layout.tsx
import { AntdRegistry } from '@ant-design/nextjs-registry'
import Providers from '@/components/Providers'

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <AntdRegistry>
          <Providers>{children}</Providers>
        </AntdRegistry>
      </body>
    </html>
  )
}
```

```tsx
// components/Providers.tsx
'use client'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import { useState } from 'react'

export default function Providers({ children }: React.PropsWithChildren) {
  const [queryClient] = useState(() => new QueryClient({
    defaultOptions: { queries: { staleTime: 60 * 1000 } },
  }))
  return <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>
}
```

**分頁 + 篩選 Hook（配合 Ant Table）：**

```ts
// hooks/useTableData.ts
import { useQuery, keepPreviousData } from '@tanstack/react-query'

export function useTableData(params: { page: number; pageSize: number; filters: Record<string, string> }) {
  return useQuery({
    queryKey: ['table-data', params],
    queryFn: () => fetchTableData(params),
    placeholderData: keepPreviousData,
  })
}
```

**Optimistic Update（送出後立即更新 UI，失敗再 rollback）：**

```ts
const mutation = useMutation({
  mutationFn: updateRecord,
  onMutate: async (newData) => {
    await queryClient.cancelQueries({ queryKey: ['table-data'] })
    const snapshot = queryClient.getQueryData(['table-data'])
    queryClient.setQueryData(['table-data'], (old) => patch(old, newData))
    return { snapshot }
  },
  onError: (_err, _vars, ctx) => {
    queryClient.setQueryData(['table-data'], ctx?.snapshot)
  },
  onSettled: () => queryClient.invalidateQueries({ queryKey: ['table-data'] }),
})
```

### 4-2 Zustand — 引端本地狀態

**職責**：純前端 UI 狀態，不涉及 API 呼叫

**典型使用情境：**
- 側邊欄 collapsed / expanded
- 跨頁選取的 row keys
- 全局 Modal / Drawer 開關狀態
- 使用者偏好（主題、語言）

```ts
// store/ui.store.ts
import { create } from 'zustand'

interface UIState {
  sidebarCollapsed: boolean
  selectedRowKeys: React.Key[]
  toggleSidebar: () => void
  setSelectedRowKeys: (keys: React.Key[]) => void
}

export const useUIStore = create<UIState>((set) => ({
  sidebarCollapsed: false,
  selectedRowKeys: [],
  toggleSidebar: () => set((s) => ({ sidebarCollapsed: !s.sidebarCollapsed })),
  setSelectedRowKeys: (keys) => set({ selectedRowKeys: keys }),
}))
```

**搜尋 / 篩選參數狀態（搭配 TanStack Query）：**

```ts
// store/table-filter.store.ts
import { create } from 'zustand'

interface FilterState {
  page: number
  pageSize: number
  filters: Record<string, string>
  setPage: (page: number) => void
  setFilters: (filters: Record<string, string>) => void
  reset: () => void
}

export const useFilterStore = create<FilterState>((set) => ({
  page: 1,
  pageSize: 20,
  filters: {},
  setPage: (page) => set({ page }),
  setFilters: (filters) => set({ filters, page: 1 }),
  reset: () => set({ page: 1, pageSize: 20, filters: {} }),
}))

const { page, pageSize, filters } = useFilterStore()
const { data, isLoading } = useTableData({ page, pageSize, filters })
```

### 4-3 職責劃分原則

| 狀態類型 | 工具 | 範例 |
|------------|------|------|
| 伺服器資料 | TanStack Query | 列表、詳情、搜尋結果 |
| 表單 draft 狀態 | React Hook Form 內部 | 未送出的表單資料 |
| 引端 UI 狀態 | Zustand | sidebar、modal、選取項 |
| URL 狀態 | Next.js `searchParams` | 可書簽的篩選、頁碼 |

## 五、補強建議

### 必補

| 補強項目 | 建議技術 | 理由 |
|----------|----------|------|
| 伺服器狀態 | `TanStack Query` | 分頁、快取、Optimistic Update 統一管理 |
| 引端 UI 狀態 | `Zustand` | 輕量，取代 Redux，職責清晰 |
| Ant Design SSR 樣式 | `@ant-design/nextjs-registry` | 避免 App Router 首屏樣式閃爍 |
| Runtime 型別驗證 | `Zod` | API schema 驗證，補齊 runtime 缺口 |

### 視需求補

| 補強項目 | 建議技術 | 適用情境 |
|----------|----------|----------|
| 複雜表單管理 | `React Hook Form` | 多步驟、複雜驗證、可測試封裝 |
| 大型表格效能 | Ant Table `virtual` 屬性 或 `@tanstack/react-virtual` | 超過 1 萬筆 row |
| DOM-heavy 元件隔離 | `next/dynamic({ ssr: false })` | 圖表、地圖、編輯器、拖曳模組 |

## 八、結論

1. **表格統一走 Ant Table**，維護單一 API 與樣式體系。
2. **TanStack Query 負責伺服器狀態**，包含分頁、快取、Optimistic Update。
3. **Zustand 負責引端 UI 狀態**，不混入伺服器資料邏輯。
4. **SSR / Hydration 問題要分成兩類處理**：Ant Design 樣式注入與 server/client 輸出一致性。
5. **Next.js App Router 專案應建立明確的 client/server 邊界規則**，避免日後擴充時反覆出現 hydration mismatch。

> 下一步建議：決定 Node.js 的角色（Route Handlers vs 獨立服務），並建立 `useIsClient`、`Providers.tsx`、`AppTable.tsx` 三個基礎骨架元件。
